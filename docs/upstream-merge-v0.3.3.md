# v0.3.3 上游合并差异

本文档记录本次合并候选分支与上游仓库的主要差异，供测试验收后再决定是否提交、打 tag、推送和创建 GitHub Release。

## 合并范围

- 上游仓库：`basketikun/infinite-canvas`
- 上游分支：`upstream/main`
- 上游提交：`8aa3fad`
- 本地合并前提交：`b188106`
- 共同基线：`e740da6`
- 提交差异：上游领先 60 个提交，本地领先上游 25 个提交。
- 本次处理方式：创建合并候选分支，手工吸收兼容能力，解决关键冲突并更新文档与版本文件；不提交、不打 tag、不推送、不创建 Release。

## 本次已实装的上游能力

- 根目录 Markdown 文档体系：按本地原有文档体系恢复 `docs/*.md`，不采用上游新文档站结构。
- 参考素材公开访问：后端保留 `/api/v1/media/references` 上传接口和 `/api/media/references/:id` 公开访问入口，第三方模型可通过 `PUBLIC_BASE_URL` 拉取本地上传的参考素材。
- 配置弹窗升级：配置弹窗接入图片、视频、文本、音频四类默认模型；视频参数和音频参数直接复用上游新增设置面板。
- 音频独立渠道：配置新增 `audioChannelId`，全局配置弹窗和画布配置节点里的音频模型选择会保存独立渠道 ID，音频生成请求按音频模型选择本地渠道或后台渠道。
- 画布音频节点：画布工具栏新增音频节点；支持本地上传、拖入、替换、播放、下载、刷新后 URL 恢复；音频节点底部生成面板可调用 OpenAI 兼容 `/audio/speech`。
- 画布图片工具栏：吸收上游图片悬浮工具栏配置能力，支持复制提示词、替换图片、锁定/自由比例、裁剪、局部遮罩编辑、图片拆分、图片放大、反推提示词、查看大图和自定义快捷工具显示。
- 局部遮罩编辑：遮罩弹窗不再自动执行生成，而是像反推提示词一样创建“局部重绘提示词”文本节点和“局部重绘配置”节点；用户需要在配置节点中手动确认后再执行，执行时会把同尺寸 PNG `mask` 作为 FormData 字段传给 OpenAI 兼容 `/images/edits`。
- 图片拆分：接入上游切图能力，图片节点工具栏可按行列拆分图片，生成多个图片子节点并自动连线。
- 图片放大：使用浏览器端插值算法生成 1K/2K/4K 放大图，不调用 AI 超分接口，结果作为右侧新图片节点插入画布。
- 反推提示词：从图片节点创建预置反推提示词文本节点，并自动创建文本模式生成配置节点，方便用户手动发起反推。
- 画布资源识别：配置节点上游输入现在能识别图片、文本、视频和音频；视频/音频可作为 Seedance 参考素材参与视频生成请求体组装。
- Seedance Agent Plan 视频链路：视频生成默认模型切到 `doubao-seedance-2.0`，创建任务走 `POST /contents/generations/tasks`，轮询走 `GET /contents/generations/tasks/{id}`，完成后读取 `content.video_url` 并只记录远程 URL，不自动下载大文件。
- Seedance 多媒体参考：视频创作台和画布配置节点均可收集图片、视频、音频参考素材，本地素材会先上传到 `/api/v1/media/references` 获得公网 URL，再写入 Seedance `content`。
- “我的素材”音频类型：新增音频素材创建、筛选、预览、下载、导入导出、保存画布音频节点和账号素材同步；对象存储生命周期扫描会把音频 storageKey 纳入保护与清理。
- Docker/SQLite 修正：保留上游对 Docker 环境下 SQLite DSN 自动归一到 `/app/data` 的处理，并新增 `PUBLIC_BASE_URL` 配置说明。

## 保留的本地差异

- 配置弹窗渠道模型选择：本地配置弹窗不再直接点击“拉取”获取渠道模型，而是复用后台管理的“选择渠道模型”弹窗，在弹窗内拉取和勾选模型。
- 视频创作台 UI：继续保留本地按生图工作台升级后的全部成果、生成结果、进行中任务、失败详情、多任务并行、底部创作工作台、本地历史记录和账号视频历史同步。
- 视频 URL 策略：视频生成成功后默认只记录 AI 返回的远程 URL，不自动下载并写入本地或服务端存储；用户可手动下载到本地或手动同步到对象存储。
- 视频对象存储同步：视频结果卡片保留同步按钮，可将 AI 临时 URL 转存到 S3/R2 对象存储并替换为云端 URL，同时继续显示 URL 来源状态。
- 账号同步：继续保留模型配置、存储配置、画布数据、生图历史、视频历史、“我的素材”和创作工作流相关同步入口。
- AI 调用日志：继续保留本地 JSONL AI 调用日志、后台查询/清理、视频最终任务响应记录和错误摘要清洗。
- 后台渠道与计费：继续保留本地用户、算力点、公开模型配置、渠道模型选择、失败返还和管理后台 AI 日志能力。
- 创作工作流：继续保留公开/个人工作流、多图工作流和 AI 创建工作流能力。
- 文件存储生命周期：继续保留素材删除时对本地 IndexedDB 和 `server:` 云端对象的引用扫描与物理删除保护。
- AI 请求失败重试：本地新增失败重试次数配置，默认 0 次；画布配置节点、生图工作台和视频创作台均可单独设置，最多 5 次。
- R2/S3 上传重试：对象存储 PUT 请求遇到超时、连接重置或 R2 5xx 时会自动重试，延迟按 100ms、300ms、900ms 递增。
- 版本更新弹窗修正：检查更新时会比较语义版本，避免远端旧版本覆盖本地更高版本，同时合并本地和远端版本记录。

## 关键冲突处理

- 前端视频、生图、画布和配置 store 以本地实现为主，补入上游音频、Seedance 参数面板、Ark Plan Base URL 归一化和 Agent Plan 请求路径，避免覆盖本地视频创作台 UI、账号同步和对象存储同步。
- 后端 `config`、`router` 合并上游 `PUBLIC_BASE_URL` 和参考素材路由，同时保留本地用户配置、对象存储、AI 日志和工作流路由。
- 文档冲突按本地根目录 Markdown 体系处理，不采用上游新文档站结构；功能说明、待测试、TODO、数据库、接口响应和合并差异均保留在 `docs/*.md`。
- 版本文件仅提升到 `v0.3.3` 候选，等待测试验收后再提交和发版。

## 未完整合并或暂未实装的上游能力

- WebDAV 同步暂未实装：上游新增的 `webdav-proxy`、WebDAV 配置项和通用 `app-sync` 服务会和本地账号同步、对象存储同步、删除生命周期、冲突解决策略重叠；当前缺少多端冲突和删除传播规则，直接接入容易造成数据覆盖或重复同步，因此本次不实装。
- AI 超分未接入：上游只有工具栏入口和占位提示，没有完整的模型渠道、计费、任务状态、存储生命周期和失败返还链路；本次保留浏览器端图片放大，不接入真正 AI 超分。
- 新文档站结构不采用：用户已明确要求恢复原有文档体系，因此不使用 `docs/content/docs/...`、文档站 Next.js 入口和文档镜像工作流。
- 上游完整工具栏逻辑没有原样覆盖：本地保留现有画布节点批量生成、历史、同步和图片组逻辑，只挑选接入了能与本地流程兼容的图片工具。

## 验收重点

- 视频创作台：文本生成、图片参考、视频参考、音频参考、多任务并行、任务完成后卡片状态、视频播放、下载到本地、同步到 R2、单条删除和账号历史同步。
- Seedance 请求参数：确认默认模型 `doubao-seedance-2.0`、`ratio`、`resolution`、`duration`、`generate_audio`、`watermark` 和多媒体参考素材 URL 按配置传递。
- 画布视频节点：生成结果能正常插入节点，远程 URL 不被强制下载，必要时可保存为素材或同步对象存储。
- 上游新增音频和参考素材：音频节点、音频生成、音频素材、参考素材上传和 `PUBLIC_BASE_URL` 公网访问提示能在本地/后台渠道下工作。
- 画布图片工具栏：裁剪、局部编辑、切图、放大、反推提示词、自定义快捷工具、复制提示词和查看大图都能按当前节点状态工作。
- 局部重绘：点击遮罩编辑后应只创建提示词节点和配置节点，不应自动发起 AI 请求；手动运行配置节点后应正常计时并展示生成结果。
- 失败重试：画布配置节点、生图工作台、视频创作台设置重试次数后，创建任务或轮询请求失败时会按设置次数重试；默认 0 次。
- 对象存储上传：国内服务器到 Cloudflare R2 网络抖动时，PUT 请求遇到超时、连接重置或 5xx 应自动重试。
- 版本更新弹窗：本地版本高于远端版本时不应被远端旧版本覆盖，版本记录应合并展示。
- 文档体系：README、`docs/index.md` 和所有说明文档应指向根目录 Markdown，不再把新文档站作为主要入口。
- Docker/SQLite：Docker 部署时数据库仍落在挂载的 `/app/data`，后台模型配置不会读到空库。
