# 待测试

## 纯生图分支与 PR #43 合并

- 后台模型渠道和配置弹窗本地渠道需要确认新增 OpenAI/Gemini 协议选择；Gemini 渠道需要确认模型列表拉取成功。
- Gemini 本地渠道需要确认文本问答、文生图和参考图图生图可用；云端 Gemini 渠道需要确认仍经过后端代理、AI 日志和算力点消费/返还。
- Gemini 蒙版局部重绘需要确认直接提示“不支持蒙版编辑”，不会自动降级到 `/responses`。
- 画布配置节点和提示词浮层需要确认生成中按钮切换为“停止”；点击停止后请求取消，节点恢复空闲状态，不进入失败态。
- 画布文本生成、图片生成、参考图生成和反推提示词需要分别确认停止生成不清除已经成功返回的结果。
- 本地 `canvas-agent` 需要确认可按 `canvas-agent/README.md` 启动，并在画布助手 Agent 模式中填写地址和 token 后连接成功。
- 本地 `canvas-agent` 需要确认使用仓库内置版本启动后，Agent 执行 `canvas_generate_image` 或 `canvas_run_generation` 时，网页侧确认后会自动触发配置节点开始生图。
- 本地 `canvas-agent` 需要确认新建空对话后第一次发送消息不会再出现 `not materialized yet`、`no rollout found` 或 `thread not found` 错误，历史面板读取空会话时应显示为空而不是出错。
- 本地 `canvas-agent` 需要确认对话运行中发送按钮会切换为停止按钮，点击后当前 Codex 对话会停止且面板恢复可输入状态。
- 网页 Agent Loop 需要确认可读取当前画布、读取选区、创建文本节点、创建生图流程、选择节点和触发文本/图片生成。
- Agent 工具列表和提示词需要确认不出现视频、音频、Seedance 或 WebDAV 能力。
- 分支 `codex/video-audio-upstream-v0.3.3` 需要确认已经保存并推送当前视频/音频合并成果。
- 当前 `main` 分支需要确认顶部导航、画布工具栏、配置弹窗、我的素材、素材库和 API 不再出现视频/音频入口。
- 生图底部工作台需要在桌面端确认“开始创作”按钮固定在右侧操作区；移动端确认按钮不遮挡、不溢出。
- 文本生图、参考图生图、局部重绘、图片拆分、图片放大、反推提示词和图片保存素材需要逐项验证。
- 生图工作台需要用 4K 或 10MB 级别图片复测 `/images/edits` 图生图，确认参考图上传不再因为大图 base64 转换报 `Maximum call stack size exceeded`。
- 生图工作台大图下载需要确认下载按钮保存的是完整原图 Blob，文件大小和直接打开云端链接下载的文件一致或仅存在正常响应头差异。
- 生图工作台生成结果卡片需要确认成功结果会展示实际图片宽高和文件体积。
- 生图工作台历史/全部结果卡片需要确认标签区会展示首张图片文件体积，多图结果会额外展示总文件体积。
- 生图工作台结果卡片和历史记录点击“重试”需要确认会复用原记录的渠道、模型、尺寸、质量、接口模式、格式、审核、超时、流式和失败重试配置。
- 生图工作台底部工作台尺寸选项需要确认已包含 1:1、16:9、9:16 的 4K 尺寸，选择后能按对应参数发起请求。
- Agnes Image 2.1 Flash 渠道需要确认默认请求体不再携带额外 seed；如果仍返回 422，需要结合渠道请求日志确认是否为渠道侧参数限制。
- 画布反推提示词需要确认 `/chat/completions` 流式返回后文本结果节点能展示最终内容，配置节点和结果节点耗时不再一直为 0，且不会在已返回内容后因为尾部残缺流块标记失败。
- 画布局部重绘需要确认只创建提示词节点和配置节点，不自动发起请求；手动运行后能正确计时并展示结果。
- 画布局部重绘需要确认遮罩图通过 `/images/edits` 随原图一起提交，不跨协议降级到 `/responses`。
- 配置弹窗“选择模型”需要确认和后台管理一样走 `/api/admin/settings/channel-models` 拉取渠道模型。
- 生图工作台生成卡片需要确认展示渠道名称，失败重试次数默认 0 且按配置生效。
- 云端渠道生图需要确认未配置失败重试时只发起 1 次 AI 请求；配置重试次数后才按次数重试。
- `/responses` 流式生图需要确认 HTTP 200 但流内出现 `event:error`、`response.failed` 或没有最终图片时会返还算力点，AI 调用日志扣点显示为 0，并按失败重试配置继续重试。
- Responses API 流式返回需要确认能从最终 completed 事件、`response.output`、base64、data URL 或 HTTP 图片 URL 中正确识别图片，不再把已返回图片的 SSE 文本误判为失败。
- 云雾等 `/images/generations`、`/images/edits` 渠道需要确认返回体包含 `b64_json` 或图片 URL 时，即使 JSON 包装被中转站截断或 HTTP 状态非 2xx，前端仍能优先提取图片结果。
- 云雾等耗时超过 60 秒的图片请求需要在服务器 openresty/Nginx 部署环境下复测，确认前端生成卡片不再保存或展示 `504 Gateway Time-out` HTML，而是等待后端最终图片结果。
- univibe 等 `/responses` 渠道需要确认单行 SSE、`response.image_generation_call.partial_image` 和 `partial_image_b64` 能被识别为图片结果；后端 AI 日志和算力点结算不再把这类已返回图片的调用记为失败。
- 上游返回 HTTP 图片 URL 的渠道需要确认生成卡片可以直接展示结果，不再错误包成 base64。
- 算力点日志需要确认 AI 消费和失败返还备注显示模型与渠道信息；失败请求应出现等额消费和返还，成功请求只保留消费。
- 所有带顶部账户区的用户页和后台页需要确认头像旁会显示当前算力点余额，余额变化后刷新用户信息可同步更新。
- “我的素材”文本卡片需要确认长提示词默认收起，详情仍可查看全文。
- 页面右上角需要确认已移除文档跳转按钮。
- R2/S3 图片上传需要确认超时、连接重置或 5xx 时会按 100ms、300ms、900ms 递增延迟重试。
- Sub2 图片和对话渠道需要确认不再固定依赖 `img.94576354.xyz` / `api.94576354.xyz`，前端渠道配置或 `SUB2_IMAGE_BASE_URL` / `SUB2_CHAT_BASE_URL` 可切换到其他兼容接口；未配置地址时应提示补充接口地址。
